MQTT, Push Data to the Device

This documentation helps you to push data to the device. Hence, it will be parsed and be a control command of the device. If you still need any assistance or found any misinformation, don't hesitate to mail us at support@antares.id

Prerequisites

  1. Having MQTT FX Software. If you dont have it, please visit this link. MQTT FX.

Steps

  1. Login to your Antares account through Antares Console https://console.antares.id

  2. Write down your access key for latter use. Get your Access Key under the Account -> Access Key

  3. Create an Application, if you already have one, then skip to the next process.

  4. Create a Device, if you already have one, then skip to the next process.

  5. Open the MQTT FX Software. The UI is shown as below figure.

  6. Firstly, You need to config so that you can connect to our MQTT Broker. The configuration can be found below.

  7. Once you successfully connected, the green icon at the top right will appear.

  8. Click the Subscribe, then start to subscribe a Topic with the following format:

    /oneM2M/resp/antares-cse/your-access-key/choose-a-serialization-format
    The choose-a-serialization-format parameter can have 2 values:

    • xml for XML format
    • json for JSON format
    This step means you will get a message/notification when there is any changes or event occur to your account. In the current tutorial, we use the MQTT FX tools to subscribe but in the real case scenario, it would be the devices which subscribe to that topic.

  9. This step, we would emulate a request that came from the mobile app. It also uses MQTT FX as the tool. Start by publishing to this topic :

    /oneM2M/req/your-access-key/antares-cse/choose-a-serialization-format
    The choose-a-serialization-format parameter can have 2 values:

    • xml for XML format
    • json for JSON format


    10. Example
    Topic: /oneM2M/req/your-access-key/antares-cse/json
    Payload 

    {
  "m2m:rqp": {
    "fr": "your-access-key",
    "to": "/antares-cse/antares-id/application-name/device-name",
    "op": 1,
    "rqi": 123456,
    "pc": {
      "m2m:cin": {
        "cnf": "message",
        "con": "on"
      }
    },
    "ty": 4
  }
}
    All the data inside the "con" is determined by yourself. You can create your own data format.


    Congratulations!! You just got your first request using MQTT API.
