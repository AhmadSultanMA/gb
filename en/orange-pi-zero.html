Application of Orange Pi Zero in Data Communication

This Documentation explains about application of Orange Pi Zero in Data Communication using HTTP RestFUL API dan the integration of Orange Pi Zero with Antares platform. If you still need any assistance or found any misinformation, don't hesitate to mail us at support@antares.id

    • Card Formatting
    • Adding Operating System

Introduction

Orange Pi is an open source single-board computer. Orange Pi has the same working system with the other varieties of Pi, like Raspberry Pi, Nano Pi, etc. Orange Pi can be operating on Android, Ubuntu, Debian, dan Raspbian sytem.

Figure 1. Orange Pi Zero

Operating System Installation

Card Formatting

Provide microSD card which will be used. Before that, the card must be formatted. Steps in formatting the card are as follows:

1. Right-click the directory of the used microSD card, then choose format menu.

Figure 3. Format Choice

2. Make sure the menu format have the configuration as in the figure below. Then, click start.

Figure 4. Format Menu Configuration

Adding Operating System

Before going to the next step, download Win32DiskImager application, then download image file at https://www.armbian.com/orange-pi-zero/ with the choice "bionic" and extract it. Steps in installing the operating system are as follows:

1. Open Win32DiskImager application

2. Browse the extracted image file. Make sure the device showed in Win32DiskImager is the MicroSD card (it is usually automaticly showed).

Figure 5. Win32DiskImager View

3. Click "Write"

Initial Configuration

To access Orange Pi Zero, user can connect the Orange Pi Zero into the computer (using USB to TTL or FTDI; connect the RX, TX, dan GND) and using PUTTY application. After connected, steps which must be done are

1. Check the port used by Orange Pi Zero with Device Manager

Figure 6. Device Manager View

2. Configure PUTTY application as follows:

Figure 7. PUTTY Configuration

3. After starting the connection, program will require the user to enter the username and password. Username used is “root”, while the password is 1234. After that, user will be required to enter the new password.

Figure 8. Changing the Password

4. After that, user will be required to enter the information needed. Please enter the information needed, you may skip it with pushing ENTER.

Figure 9. Entering the Information Needed

5. The general view of Orange Pi Zero is:

Figure 10. Orange Pi General View

6. Update the OS with

sudo apt-get update

sudo apt-get upgrade

Python Programming

This section will provide tutorial about room temperature and humidity sensing with DHT22 sensor, Orange Pi Zero, Antares, and HTTP RestFUL API

1. First, install python dan “pip” with:

sudo apt-get install python

sudo apt-get install python-pip

2. After that, user can get the base for programming with:

git clone https://github.com/ionutpi/DHT22-Python-library-Orange-PI

3. To simplify the execution of the program, program contained in the folder, which are cloned from the web, can be copied to home with:

cp /root/DHT22-Python-library-Orange-PI/dht22.py ~

4. Checking whether the program can be executed can be done with (make sure the sensor is in right port):

python dht22.py

5. Results received are as follows:

Figure 11. Sensor Reading Results

6. After that, open POSTMAN application (Please check the tutorial STORE DATA OF A PARTICULAR DEVICE). After filling the information needed (in JSON), click "Code" menu.

Figure 12. "Code" Choice on POSTMAN Application

7. Choose the language used in the programming (know your version of Python with "python -V"). Other programming language can be used (as long as it's available).

Figure 13. Code choice

Figure 14. Code for Sending Data in Python Language

8. Program obtained is copied and pasted to new program file and edited. The aim is to obtain temperature and humidity data every 1 second and then send it to Antares. We get:


from pyA20.gpio import gpio
from pyA20.gpio import port
 
#import RPi.GPIO as GPIO
import dht22
import time
import datetime
import requests 
# initialize GPIO
#gpio.setwarnings(False)
#gpio.setmode(GPIO.BCM)
PIN2 = port.PA6
gpio.init()
#gpio.cleanup()
 
 
# read data using pin 14
instance = dht22.DHT22(pin=PIN2)
 
while True:
    result = instance.read()
    if result.is_valid():
        print("Last valid input: " + str(datetime.datetime.now()))
        print("Temperature: %.2f C" % result.temperature)
        print("Humidity: %.2f %%" % result.humidity)
 
    time.sleep(1)
	
	url = "https://platform.antares.id:8443/~/antares-cse/antares-id/SmartCoba/SmartS1"

	payload = "{\r\n  \"m2m:cin\": {\r\n    \"con\": \"{\\\"key1\\\":"+str(datetime.datetime.now())+", \\\"key2\\\":\\\""+str(result.temperature)+"\\\", \\\"keyN\\\":\\\""+str(result.humidity)+"\\\"}\"\r\n  }\r\n}"
	headers = {
		'X-M2M-Origin': "access-id:access-password",
		'Content-Type': "application/json;ty=4",
		'Accept': "application/json",
		'Cache-Control': "no-cache",
		'Postman-Token': "0314aaab-c1bf-4b4c-90c8-7b2865e26965"
		}

	response = requests.request("POST", url, data=payload, headers=headers)

	print(response.text)

If when the program is executed and there's an error of not finding the particular Python module (library), user can download the library with:

pip install “module_name”
