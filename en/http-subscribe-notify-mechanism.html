HTTP, Subscribe Device and Get Notification

This documentation helps you on How to Subscribe a Device and get a Notification of an Event. If you still need any assistance or found any misinformation, don't hesitate to mail us at support@antares.id

Prerequisites

  1. Having NGROK Software. If you dont have it, please visit this link. NGROK.
  2. Having Node JS. If you dont have it, please visit this link. Node JS.
  3. Having POSTMAN. If you dont have it, please visit this link. POSTMAN.

Steps

  1. Login to your Antares account through Antares Console https://console.antares.id

  2. Create an Application, if you already have one, then skip to the next process.

  3. Create a Device, if you already have one, then skip to the next process.

  4. Start to write a code to catch data from ANTARES. You can use the codes below: Copy it to your local computer then save it with .js extension. In this tutorial we saved it with monitor.js 
    var express = require('express');
var app = express();
var server = require('http').createServer(app);
var bodyParser = require('body-parser');
var util = require('util')

app.use(bodyParser.json());
app.use(bodyParser.urlencoded({ extended: false }));

app.post('/monitor', function(req, res, next) {
  console.log(util.inspect(req.body, false, null));  
  res.send('ack');
});


server.listen(9000, function() {
  console.log('App listening on port 9000');
});


  5. Open the command prompt (windows) or terminal (linux)

    Go to the path of the code you saved before

  6. Run the code by doing as figure below.

    The successfully run code can be seen as figure below.

    Having trouble? If yes then execute the syntax below. If no, skip this process.

    Install the uninstalled packages.



    Please do Allow Access while running the code.

  7. Start running the NGROK. NGROK will help you to expose your localhost to a public domain. Firstly go to your NGROK path. Make sure you have extracted it so it is already an .exe file. Run the NGROK by forwarding the 9000 port.

  8. Open the device you created before. Click Subscribe button and there will be a pop up menu.



    Please add the /monitor at the end of the NGROK url as figure below.

    Once you successfully subscribed it, a pop up will appear.

  9. Let's check whether your server get a notification by emulating a device using POSTMAN. Please open the POSTMAN, and fill all the parameter with the following informations.

    Field Value
    URL https://platform.antares.id:8443/~/antares-cse/antares-id/your-application-name/your-device-name
    or
    https://platform.antares.id:8443/~/antares-cse/your-device-ID

    Note: your-device-ID must be starting with CNT. e.g. CNT-842419289
    Method POST
    Header Key Value
    X-M2M-Origin access-id:access-password
    Content-Type application/json;ty=4
    Accept application/json
    Body 
    {
  {
  "m2m:cin": {
    "xmlns:m2m": "http://www.onem2m.org/xml/protocols",
    "cnf": "application/json",
    "con": "{\"temperature\":30.3, \"humidity\":35.1}"
  }
}

  10. Once you sent the data, your application server monitor will get the exact data that the server get. The data can be seen as follows.

    11. Congratulations!! You just got your first notification data.
